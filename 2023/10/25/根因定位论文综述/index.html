Papers:

  1. Causal Inference-Based Root Cause Analysis for Online Service Systems with Intervention Recognition - Tsinghua University, BizSeer (2022)

Proposed Model:
CIRCA - https://github.com/NetManAIOps/CIRCA

  • Unsupervised
  • Monitoring data
  • CIRCA 的核心思想是监控变量成为根本原因指标的充分条件，即在因果贝叶斯网络（CBN）中，条件概率分布的变化。
  • CIRCA 在在线服务系统的应用中，根据系统架构的知识和一组因果假设在监控指标之间构建图。

Compared models:

  1. Graph-Based Root Cause Localization in Microservice Systems with Protection Mechanisms - Beijing University of Posts and Telecommunications School of Computer Science (2022)

Proposed Model:
MicroGBPM
- Unsupervised*
- Monitoring data, trace data
- 提出了一个名为MicroGBPM的新框架，用于定位异常服务的根本原因，该框架考虑了保护机制的影响。
- 当发生故障时，将服务之间的异常传播建模为一个动态构建的、带有指标和追踪信息的服务属性图。为了消除保护机制的影响，设计了一种两阶段的动态校准策略来调整服务之间异常传播的概率。然后提出了一种随机游走方法，使用PageRank算法来计算根因结果。

Compared models:

MicroHECL

  1. Incremental Causal Graph Learning for Online Root Cause Analysis - NEC Laboratories America (2023)

Proposed Model:
CORAL
- Unsupervised
- Monitoring data
- CORAL，一种在线RCA框架，能够自动触发RCA过程并递增地更新RCA模型。CORAL包括触发点检测、递增解耦因果图学习和基于网络传播的根因定位三个部分。
- 触发点检测部分的目标是自动并近实时地检测系统状态转换。为了实现这一目标，开发了一种基于多变量奇异谱分析和累积和统计的在线触发点检测方法。
- 为了高效地更新RCA模型，我们提出了一种递增解耦因果图学习方法，用于解耦状态不变信息和状态依赖信息。
- 之后，CORAL应用带重启的随机游走到更新的因果图上，以准确地识别根因。在线RCA过程在因果图和生成的根因列表收敛时终止。

  1. Interdependent Causal Networks for Root Cause Localization - NEC Laboratories America Inc., University of Central Florida, Amazon Inc., University of Utah (2023)

Proposed Model:
REASON
- Unsupervised
- Monitoring data
- REASON用于自动发现因果关系，分为内部级别（网络内）和跨级别（网络间）。
- 扑因果发现 (TCD)：目标是模拟故障传播以追溯到根本原因。使用分层图神经网络来构建表示因果关系的相互依赖的网络。利用带重启的随机游走模型系统故障的网络传播。
- 个体因果发现 (ICD)：重点是捕捉单个系统实体的突变变化模式。分析时间序列数据，估计实体成为根因的可能性。
- 结合拓扑和个体因果得分，识别前K个可能的根本原因。

Compared models:

  1. Localizing Failure Root Causes in a Microservice through Causality Inference - Tsinghua University, Nankai University, Alibaba Group (2020)

Proposed Model:
MicroCause
- Unsupervised *
- Monitoring data
- 提出了一个框架 MicroCause，用于准确定位微服务内的故障根源监控指标。
- MicroCause 结合了一个简单有效的路径条件时间序列 (PCTS) 算法，用于捕获时间序列数据的顺序关系；
- 以及一个新颖的时间因果导向随机游走 (TCORW) 方法，整合了监控数据的因果关系、时间顺序和优先级信息。

Compared models:

Anomaly Time Order

  1. MicroHECL: High-Efficient Root Cause Localization in Large-Scale Microservice Systems - Fudan University, Alibaba Group (2021)

Proposed Model:
MicroHECL:
- Unsupervised
- Trace data
- MicroHECL在动态构建的服务调用图的基础上，分析可能的异常传播链，并基于相关性分析对候选根本原因进行排名。
- 结合机器学习和统计方法，为不同类型的服务异常（即性能、可靠性、流量）设计定制模型。采用剪枝策略来提高效率，消除异常传播链分析中不相关的服务调用。
- 实际应用：MicroHECL已在阿里巴巴使用，并在根本原因定位时间从30分钟缩短到5分钟的情况下实现了前三名的命中率为68%。

Compared models:
MonitorRank
Microscope

  1. MicroRCA: Root Cause Localization of Performance Issues in Microservices - Department of Computing Science, Umea ̊ University, Umea ̊, Sweden (2020)

Proposed Model:
MicroRCA - https://github.com/elastisys/MicroRCA
- Unsupervised
- Monitoring data
MicroRCA 通过关联应用性能症状和相应的系统资源利用率，实时推断根本原因，无需任何应用工具的帮助。根本原因定位基于一个属性图，该图模型在服务和机器之间的异常传播。

Compared models:
MonitorRank
Microscope

  1. Practical Root Cause Localization for Microservice Systems via Trace Analysis - Tsinghua University, Tianjin University, China Minsheng Bank, BizSeer, BNRist (2021)

Proposed Model:
TraceRCA - https://github.com/NetManAIOps/TraceRCA
- Unsupervised
- Trace data
- 异常追踪检测：通过无监督多指标方法检测异常调用，并自适应选择有用指标。
- 挖掘可疑微服务集：找出符合异常追踪多、正常追踪少规律的微服务集。
- 微服务排名：为可疑微服务集中的每个微服务计算分数并排名，辅助操作员定位故障。

Compared models:

Compared Models:

T-Rank：
⁃ Unsupervised*
⁃ trace data
用谱分析算法来定位异常服务的根本原因。这种方法是基于追踪数据的，通过分析异常服务与正常服务的差异来确定可能的根本原因。

TraceRank: https://onlinelibrary.wiley.com/doi/abs/10.1002/smr.2413
⁃ Unsupervised*
⁃ trace data
主要包括异常检测模块和根本原因分析模块，利用谱分析和基于 PageRank 的随机游走方法对离散的端到端追踪数据进行处理，从而识别和定位导致性能问题的异常服务。

Microscope
⁃ Unsupervised
⁃ Monitoring data, Trace data*
利用服务指标构建服务因果图，并实时推断性能问题的原因。这种方法是基于监控和指标数据的，通过分析服务之间的关系和性能指标来定位可能的问题点。

TON18 和 MonitorRank:
⁃ Unsupervised
⁃ Monitoring data
使用依赖关系图和随机游走算法来定位根本原因。

CloudRanger:
⁃ Unsupervised
⁃ Monitoring data
使用PC算法生成依赖关系图，然后使用二阶随机游走算法追踪根本原因API，不需要标注的数据

MEPFL:
⁃ Supervised
⁃ Trace data
用于预测和定位微服务应用中潜在错误的方法，它收集服务指标并在追踪和服务级别训练预测模型以预测服务故障。

TraceAnomaly:
⁃ Unsupervised
⁃ Trace data
是一种使用异常检测和服务调用图的方法来定位微服务系统中的根本原因，它依赖于对服务调用的监控和分析。
