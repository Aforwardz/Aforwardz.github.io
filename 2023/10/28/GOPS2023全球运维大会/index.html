GOPS2023全球运维大会

发表于2023-10-28|更新于2023-11-06|点滴

介绍

GOPS（GreatOps）全球运维大会是国内第一个运维行业大会，面相互联网、金融、通信及传统行业广大运维人员，旨在传播先进技术思想和理念，分享行业最佳实践。

迄今为止，GOPS已举行20次。GOPS主要面向运维行业的中高端技术人员，包括运维、开发、测试、架构师等群体。目的在于帮助IT技术从业者系统学习了解相关知识体系，让创新技术推动社会进步。

名词

  • XOps：XOps 普遍被认作是DataOps、MLOps、ModelOps 和Platform Ops 4 大类，他们分别是数据、机器学习、模型、平台等技术热点或理念趋势与运维（Ops）的融合
  • FinOps：FinOps 是“Finance”和“DevOps”的综合体，强调通信和协作业务和工程团队。也被称为“云财务管理”、“云财务工程”、“云成本管理”、“云优化”或“云财务优化”等
  • MaaS：Model as a Service 模型即服务，主要基于云的大模型服务
  • FaaS：Function as a Service 功能即服务，是一种云计算服务，允许开发人员以功能的形式来构建、计算、运行和管理这些应用包，无需维护自己的基础架构
  • 检索增强生成（RAG）：把一个信息检索组件和文本生成模型结合在一起
  • 链式思考提示（CoT）：提示通过中间推理步骤实现了复杂的推理能力

重点

  1. AIOps产品化别人做什么功能
  2. 告警降噪的方案
  3. 变更场景下的AI异常检测
  4. 根因定位都输出什么可解释的或可用的信息，根因定位后在什么场景下应用

笔记

农业银行数字化、智能化运营转型与实践

  • 趋势预测
  • 无阈值监控
  • 图计算，态势预警





XOps持续赋能企业高质量智慧化发展

  • 太顶层设计，有点需，效果不一定

运维数字化转型助力企业高质量发展

  • 指标、日志、链路可观测
  • 健康度检查


AIGC技术在统一运维领域的应用探索

  • 外挂知识库
  • 混合专家模型多模态运维知识图谱



一体化数智运维管理场景的新探索

  • 固定阈值、分段阈值、动态阈值
  • 交易码根因定位







开源工具与商业产品完美融合：真正建立一体化可观测平台

  • 维度内离群模式分析
  • 组合维度下钻剪枝
  • 启发式学习，超过3次调用集学习故障模式
  • 两阶段自适应根因分析：即横向服务定位，再纵向定位到sql、日志、错误等




分钟级故障治愈：携程AIOps建设之路

  • 异常检测：统计方法、相似性度量
  • 趋势预测：LSTM、Attention
  • 关联分析：频繁集、规则
  • 根因定位：Adtributor、HotSpot
  • 典型异常场景：点异常、阴跌识别、周期下降
  • 异常程度度量
  • 告警状态机：设置告警状态，根据历史故障模式计算状态转移概率，判断当前下一状态，从而判断是否需要告警，减少误告
  • 容量管理
  • 判断是否适合AI建模：长时间跌0等（只针对业务指标）






金融应用全链路分析的“合纵连横”

横向感知用户行为：客户进行业务交互的全流程旅程
纵向钻取调用链

国泰君安数据和算法驱动的智能运维平台建设

  • 算法服务
  • 指标异常检测：
  • 多维异常定位
  • 日志异常检测：错误日志统计，时段新增、历史新增，日志下钻
  • 系统健康度：马尔科夫链
  • 告警治理：告警分布，消除高频/周期性告警










AIOps+时代：LLM-based Agent实践







  • Agent技术

大模型在智能运维中的若干探索

  • GPT有一定能力，但开源模型差很多
  • 开源大模型差距：1. 缺乏运维知识 2. 工具交互能力弱 3. 推理力能力弱
  • 运维大模型：基于开源，训练微调，检索增强，提示词
  • 四个原则：1. 本地化部署 2. 集成现有工具 3. 不能为了LLM而LLM 4. 充分发挥LLM的长处
  • 修改参数：预训练 -> 全参数微调 -> 不分参数微调
  • 不修改参数：1. 提示词 2. 指令 3. 检索增强 4. Agent&Tool

应用：

  • 基于知识增强的告警理解
  • 日志/告警解析
  • 运维数据探查：text2query_language(sql/es/cypher)
  • 融合大模型的告警分析
  • 故障分析
  • 告警分析与根因定位














蚂蚁集团变更管控实践

  • 变更：是对线上服务“状态”的修改，由内部人员发起
  • 金丝雀变更模式
  • 变更智能化：
    • 时序指标监控：3-sigma、kde
    • 变更前后对比、变更组与非变更组对比、变更组与全局对比
    • 日志堆栈异常检测：新增、突增异常 – 离线训练、在线预测
    • 链路异常检测：trace日志聚合+结合中间件
    • 配置值自适应校验
  • 开源社区：AlterShield
